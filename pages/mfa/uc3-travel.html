MFA-UC3: Impossible Travel Detection

User logs in from Toronto, then Moscow 30 minutes later. Unless they have a teleporter, this is credential theft. Calculates geographic distance and travel time to detect physically impossible login sequences.

Severity HIGH
MITRE T1078
Schedule Every 15 min
Time Range Last 4 hours

How Impossible Travel Detection Works

The Math Behind It

1. Get two consecutive logins for the same user with different locations

2. Calculate geographic distance between locations (using lat/lon)

3. Calculate time elapsed between logins

4. Calculate required travel speed = distance ÷ time

5. If speed > 800 km/h (commercial flight speed) → Impossible Travel

Step 1: Decision Tree (Detection Logic)

flowchart TD
    START[Successful login event received] --> A[Get user and location from current login]
    A --> B[Lookup previous login for same user within 4 hours]
    B --> C{Does previous login exist}
    C -->|No| FIRST[First login in window - no comparison possible]
    C -->|Yes| D[Get location of previous login]
    D --> E{Are locations different}
    E -->|No| SAME[Same location - no travel detected]
    E -->|Yes| F[Calculate distance between locations using lat and lon]
    F --> G[Calculate time elapsed between logins]
    G --> H[Calculate required travel speed]
    H --> H1[speed_kmh = distance_km divided by hours_elapsed]
    H1 --> I{Is speed GT 800 km per hour}
    I -->|No| J{Is speed GT 500 km per hour}
    J -->|No| POSSIBLE[Possible by fast travel - low priority]
    J -->|Yes| SUSPICIOUS[Suspicious travel speed - needs review]
    I -->|Yes| IMPOSSIBLE[IMPOSSIBLE TRAVEL - physically cannot be legitimate]
    IMPOSSIBLE --> K{Is either location a known VPN exit}
    K -->|Yes| VPN[Possible VPN usage - reduce severity]
    K -->|No| L{Is user privileged}
    L -->|Yes| CRIT[CRITICAL - Privileged user impossible travel]
    L -->|No| HIGH1[HIGH - Standard user impossible travel]
    VPN --> MED[MEDIUM - Possible VPN but verify]
    SUSPICIOUS --> MED
    CRIT --> CREATE[Create Notable Event]
    HIGH1 --> CREATE
    MED --> CREATE
    CREATE --> ENRICH[Enrich with both locations and timeline]

Step A: Build SPL in Search First

Where: Splunk Search → Build and test SPL

CyberArk Identity - Impossible Travel Detection

PRIMARY: For privileged users and admins

index=cyberark sourcetype=cyberark:identity:audit earliest=-4h
| where EventType IN ("Cloud.Core.Login", "Cloud.Core.MfaAuthentication") AND Result="Success"
| eval user = Username
| eval src_ip = FromIPAddress
| eval city = City
| eval country = Country
| eval lat = Latitude
| eval lon = Longitude
| sort 0 user _time
| streamstats current=f window=1 
    last(_time) as prev_time 
    last(src_ip) as prev_ip 
    last(city) as prev_city 
    last(country) as prev_country
    last(lat) as prev_lat 
    last(lon) as prev_lon 
    by user
| where isnotnull(prev_time) AND (city!=prev_city OR country!=prev_country)
| eval time_diff_hours = round((_time - prev_time) / 3600, 2)
| where time_diff_hours > 0 AND time_diff_hours < 4
| eval lat_rad = lat * 3.14159 / 180
| eval lon_rad = lon * 3.14159 / 180
| eval prev_lat_rad = prev_lat * 3.14159 / 180
| eval prev_lon_rad = prev_lon * 3.14159 / 180
| eval dlat = prev_lat_rad - lat_rad
| eval dlon = prev_lon_rad - lon_rad
| eval a = pow(sin(dlat/2), 2) + cos(lat_rad) * cos(prev_lat_rad) * pow(sin(dlon/2), 2)
| eval distance_km = round(2 * 6371 * asin(sqrt(a)), 0)
| eval speed_kmh = round(distance_km / time_diff_hours, 0)
| where speed_kmh > 500
| eval travel_type = case(
    speed_kmh > 800, "IMPOSSIBLE - Faster than commercial flight",
    speed_kmh > 500, "SUSPICIOUS - Very fast travel",
    1=1, "Possible"
)
| lookup privileged_users.csv userPrincipalName AS user OUTPUT is_privileged role
| eval severity = case(
    speed_kmh > 800 AND is_privileged="true", "critical",
    speed_kmh > 800, "high",
    1=1, "medium"
)
| eval time_diff_min = round(time_diff_hours * 60, 0)
| table _time user role is_privileged prev_city prev_country city country distance_km time_diff_min speed_kmh travel_type src_ip prev_ip severity

Speed Reference Guide

What Speeds Mean

Speed (km/h)EquivalentInterpretationAction
< 200 Driving / train Normal travel No alert
200 - 500 Domestic flight Possible with air travel Low priority alert if sensitive user
500 - 800 Fast international flight Suspicious but theoretically possible MEDIUM - Review
800 - 1200 Supersonic (no commercial) Impossible without VPN/proxy HIGH - Investigate
> 1200 Faster than sound Definitely credential theft or VPN CRITICAL - Immediate action

Step D: Handle False Positives

Common False Positives & Solutions

FP SourceHow to IdentifySolution
VPN usage User connects via VPN, IP geolocates to VPN server location Create VPN exit IP lookup; if one location is known VPN, reduce severity
Cloud service IPs App accessed via cloud proxy (Zscaler, Netskope), geolocates to random location Identify cloud proxy IP ranges; exclude or tag separately
Mobile roaming Mobile users crossing borders, carrier IP changes mid-session Higher tolerance for mobile user agents; check device consistency
Geolocation errors IP geolocation database has incorrect coordinates Use multiple geolocation sources; allow for error margin near borders
Shared/service accounts Multiple people using same account from different locations Flag shared accounts separately; these need different handling
