MFA Fundamentals

Understanding Multi-Factor Authentication for Detection Engineering

‚ùì WHAT is MFA?

Multi-Factor Authentication (MFA) requires users to provide two or more verification factors to gain access. This adds a critical security layer beyond just passwords.

Knowledge Factor

Password, PIN, security questions. Can be stolen via phishing.

Possession Factor

Phone, hardware token, smart card. Target of MFA fatigue attacks.

Inherence Factor

Biometrics - fingerprint, face, voice. Most resistant to remote attack.

üéØ WHY Do We Need MFA Detection?

MFA is the Last Line of Defense
When credentials are compromised (phishing, breach, purchase), MFA is often the ONLY thing preventing account takeover. Attackers know this and specifically target MFA.

Detection Strategy
  1. Count MFA denials per user in rolling window (10 min)
  2. Alert at threshold (5+ for privileged, 10+ for standard)
  3. Check for subsequent success ‚Äî this indicates compromise
  4. If denials + success: CRITICAL ‚Äî disable account immediately

MFA Methods Comparison

Method Security Fatigue Risk Notes
FIDO2 / WebAuthn Highest (5) None Hardware key, phishing resistant
Authenticator (Number Match) High (5) Low Must enter number from screen
Authenticator (Push) Medium (4) HIGH Primary fatigue attack target
TOTP App Medium (3) None Time-based codes, no push
SMS/Voice Low (2) None SIM swap vulnerable
Email OTP Lowest (1) None Email often compromised too
MFA Method Downgrade = Attack Indicator
If a user suddenly switches from FIDO2/Authenticator to SMS, this may indicate an attacker trying to use a weaker method. Monitor for method changes.
