Issue: Data Not Flowing

Check if index exists
| eventcount summarize=false index=akamai
| eventcount summarize=false index=cyberark
| eventcount summarize=false index=azure

If count = 0: Index may not exist or has no data. Check Settings → Indexes.

List all HEC tokens
| rest /services/data/inputs/http 
| search disabled=0 
| table title index sourcetype disabled

If disabled=1: Enable in Settings → Data Inputs → HTTP Event Collector

Akamai

  • Check Akamai Control Center → SIEM Integration
  • Verify API credentials not expired
  • Check Splunk add-on input status

CyberArk

  • Check CyberArk Identity → Logs → Export settings
  • Verify SIEM connector configuration
  • Check Splunk add-on input status
Check for HEC errors
index=_internal sourcetype=splunkd component=HttpInputDataHandler 
| stats count BY log_level message 
| sort - count

Issue: Wrong Timestamp

Compare _time vs raw timestamp
index=akamai sourcetype=akamai:siem earliest=-1h
| head 10
| eval parsed_time = strftime(_time, "%Y-%m-%d %H:%M:%S")
| table _time parsed_time _raw

Akamai

TIME_FORMAT = %s%3N
Epoch milliseconds

CyberArk

TIME_FORMAT = %Y-%m-%dT%H:%M:%S.%3N%Z
ISO 8601 format

Entra ID

TIME_FORMAT = %Y-%m-%dT%H:%M:%S.%6N%Z
ISO 8601 with microseconds

Issue: Missing Fields

Check field extraction
index=akamai sourcetype=akamai:siem earliest=-1h
| head 100
| fieldsummary 
| where count < 100
| table field count distinct_count
| sort - count
Nested JSON Fields
Akamai and CyberArk use nested JSON. Use | spath at the start of your search to extract all fields, or | spath path=attackData.clientIP output=clientIP for specific fields.

Issue: Lookup Not Working

Verify lookup file
``` Check if lookup exists and has data
| inputlookup privileged_users.csv
| head 10

``` Test lookup with sample data
| makeresults 
| eval user="test.admin@myfirstpro.com"
| lookup privileged_users.csv userPrincipalName as user OUTPUT is_privileged role
| table user is_privileged role

Common Issues

  • Field name case mismatch (user vs User)
  • Lookup file not in correct location
  • CSV has BOM or encoding issues
  • Missing transforms.conf definition

Solutions

  • Use lowercase field names consistently
  • Upload via Settings → Lookups → Add new
  • Save as UTF-8 without BOM
  • Create automatic lookup in transforms.conf

Issue: Alert Not Firing

Run the detection query manually in Search. If it returns results, the SPL is working.

``` Run over longer time range to verify logic
index=akamai sourcetype=akamai:siem earliest=-7d
| spath path=attackData.rules{}.ruleAction output=action
| where action="deny"
| stats count BY _time | where count > 0
| rest /services/saved/searches 
| search title="WAF*" OR title="MFA*"
| table title is_scheduled disabled cron_schedule next_scheduled_time

Check: is_scheduled=1, disabled=0, cron_schedule is valid

Verify the alert is set to trigger when "Number of results is greater than 0" (not equal to).
Also check suppression settings aren't preventing alerts.
