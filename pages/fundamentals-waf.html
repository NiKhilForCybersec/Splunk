WAF Fundamentals

Understanding Web Application Firewalls for Detection Engineering

‚ùì WHAT is a WAF?

A Web Application Firewall (WAF) is a security control that monitors, filters, and blocks HTTP/HTTPS traffic to and from a web application. It operates at Layer 7 (Application Layer) of the OSI model.

flowchart LR A[User Request] --> B{WAF} B -->|Clean| C[Web Application] B -->|Malicious| D[Block/Challenge] C --> E[Response] E --> B B --> A style B fill:#4a90c2,color:#fff style D fill:#dc2626,color:#fff

Signature-Based

Matches known attack patterns (SQLi, XSS, etc.) against request content

Anomaly-Based

Detects deviations from normal traffic patterns

Behavioral

Analyzes request sequences and user behavior

Detection Engineering Process

Step 1: Validate Data

Confirm Akamai logs are flowing to Splunk with correct field extraction. Check attackData.rules{}.ruleAction extracts properly.

Step 2: Establish Baseline

Run historical analysis over 14+ days. Calculate normal block volume patterns by hour of day, day of week.

Step 3: Build Detection

Write SPL that compares current activity against baseline. Use statistical thresholds (2œÉ) to identify anomalies.

Step 4: Tune & Deploy

Test against historical data, identify false positives, create allowlists, deploy as scheduled alert.

Common Attack Categories

SQL Injection (SQLi)

MITRE: T1190 | Severity: Critical

Attacker injects SQL code via input fields to manipulate database queries. Look for 'OR 1=1, UNION SELECT, -- in parameters.

Cross-Site Scripting (XSS)

MITRE: T1189 | Severity: High

Attacker injects malicious scripts into web pages. Look for <script>, javascript:, encoded variants in requests.

Local File Inclusion (LFI)

MITRE: T1190 | Severity: High

Attacker attempts to read local files. Look for ../../../etc/passwd, ..%2F path traversal.

Remote Code Execution (RCE)

MITRE: T1190 | Severity: Critical

Attacker executes arbitrary code on server. Look for shell commands, Log4j patterns, serialization exploits.

| where action="deny"
| spath path=attackData.rules{}.ruleTag output=category
| rex field=category "OWASP_CRS/WEB_ATTACK/(?<attack_type>[^/]+)"
| stats count BY attack_type
| sort - count
