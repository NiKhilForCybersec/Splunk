Questions for Stakeholders

Pro Tip: Document Everything
Write down answers to these questions in your notes immediately. You'll reference them throughout the engagement, and they're invaluable for handover documentation.
# Question Why It Matters Answer
1 What is the Akamai index and sourcetype? Need correct search targets
2 How is Akamai data ingested? (SIEM API / TA / HEC) Troubleshooting data flow
3 What Akamai products are in use? (KSD, Bot Manager, etc.) Determines available log types
4 What is the typical data latency? Detection window tuning
5 Are there known data gaps or outages? Baseline accuracy
6 Which CyberArk products are deployed? (Identity, PAM, both?) MFA detection design
7 Is CyberArk the PRIMARY MFA for privileged users? Critical for MFA detection
8 Is CyberArk federated with Entra ID? Correlation possibilities
# Question Why It Matters
9 Are there existing WAF or MFA detections in Splunk? Don't duplicate work
10 Are there existing dashboards to review? Understand current visibility
11 What are known false positive sources? Avoid repeating mistakes
12 Does a privileged user list exist? Where? Critical for MFA detections
13 What's the SOC workflow for WAF alerts? Runbook alignment
14 What ticketing system is used? (ServiceNow, etc.) Alert action configuration
15 Is Splunk Enterprise Security deployed? Notable events vs. standard alerts
# Question Why It Matters
16 Who is the primary client decision-maker? Approval authority
17 What's the change management process? Deployment workflow
18 Are there scheduled maintenance windows? Avoid false positive tuning issues
19 What are the top 3 security concerns for the client? Prioritization alignment
20 Are there compliance requirements (PCI, PIPEDA)? Data handling constraints
21 Is Splunk Cloud in a Canadian region? PIPEDA data residency

Day 1 Discovery Queries

Run these queries to understand what data exists in the Splunk environment.

Master Discovery ‚Äî Find All Data Sources SPL 
| tstats count WHERE index=* BY index sourcetype 
| sort - count 
| head 50
Find Akamai WAF Data SPL 
index=* sourcetype=*akamai* earliest=-1h 
| stats count by index sourcetype
| sort - count
Find Entra ID / Azure AD Data SPL 
index=* (sourcetype=*azure*signin* OR sourcetype=*azure*aad* OR sourcetype=*entra*) earliest=-1h 
| stats count by index sourcetype
| sort - count
Find CyberArk Data SPL 
index=* sourcetype=*cyberark* earliest=-1h 
| stats count by index sourcetype
| sort - count
Validate Data Freshness SPL 
| tstats latest(_time) as last_event WHERE index IN (akamai, azure, cyberark) BY index sourcetype
| eval hours_ago = round((now() - last_event) / 3600, 1)
| eval status = case(hours_ago < 1, "‚úÖ Current", hours_ago < 4, "‚ö†Ô∏è Delayed", true(), "‚ùå Stale")
| eval last_event = strftime(last_event, "%Y-%m-%d %H:%M:%S")
| table index sourcetype last_event hours_ago status
| sort status

Red Flags to Watch For

Data Not in Splunk
Discovery queries return 0 results? Escalate immediately. Data may not be onboarded. This is a critical blocker.
No Privileged User List
MFA detections require a privileged user lookup. If none exists, you'll need to build one with IAM team. Add to timeline.
Outgoing Engineer Unavailable
No handover? Document everything independently. Run all discovery queries. Ask more questions in stakeholder meetings.
Scope Creep Signals
"While you're at it..." ‚Äî document and defer politely. Scope is WAF/DDoS + MFA only. New requests go through proper channels.
Significant Data Latency
If Akamai data is > 15 minutes delayed, adjust detection time windows. Document and discuss with client.
Missing Field Extractions
Critical fields not extracted? TA may be misconfigured or missing. Check props.conf and add-on installation.
