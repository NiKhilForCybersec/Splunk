Critical Questions to Ask

Ask These on Day 1 — Document the Answers
These questions help you understand the environment and avoid mistakes.

Environment Questions

Question Why It Matters Document This
"What's the Splunk Cloud stack name/URL?" Need this to access Splunk Stack name, full URL
"Is this Victoria Experience or Classic?" UI and admin workflows differ Version type
"What indexes contain the data I'll work with?" Need to know where to search Index names and purposes
"What TAs are installed?" Know what's parsing your data List of TAs
"Is Enterprise Security installed? Version?" ES workflows differ by version ES version
"What data models are accelerated?" Know what tstats will work on Accelerated DMs

Project Questions

Question Why It Matters Document This
"What's the current project status?" Understand where things stand Status summary
"What are the immediate deliverables?" Know your priorities Next 2-week deliverables
"Who are the key stakeholders?" Know who to communicate with Name, role, contact
"What's the change management process?" Know how to deploy changes CM process steps
"Are there any known issues or blockers?" Avoid stepping on landmines Known issues list
"What was tried before that didn't work?" Don't repeat failed approaches Past attempts

Practical Questions

Question Why It Matters Document This
"How do I submit support cases to Splunk?" You'll need this for app installs, etc. Support process
"What's the naming convention for saved searches?" Follow existing patterns Naming standards
"Where do I document my work?" Knowledge management Doc locations
"Who approves enabling correlation searches?" Know the approval chain Approver names
"Is there a dev/test environment?" Don't test in production Test env details
"What are the maintenance windows?" Plan deployments accordingly MW schedule

Check data recency for your index index=YOUR_INDEX earliest=-24h | stats count by sourcetype | sort -count # 3. Check field extraction index=YOUR_INDEX | head 100 | fieldsummary | where count>0 | table field count # 4. Day 1 Printable Checklist

Morning Tasks

  • ☐ Arrive 15 minutes early
  • ☐ Set up laptop and verify network access
  • ☐ Get all required access (Splunk, email, Teams, documentation)
  • ☐ Meet team members and note names/roles
  • ☐ Get walkthrough of the environment
  • ☐ Identify your primary point of contact

Midday Tasks

  • ☐ Ask all environment questions (documented above)
  • ☐ Ask all project questions (documented above)
  • ☐ Review existing project documentation
  • ☐ Understand the current sprint/deliverables
  • ☐ Join all required meetings/channels

Afternoon Tasks

  • ☐ Log into Splunk and validate access
  • ☐ Run validation commands (above)
  • ☐ Review existing correlation searches
  • ☐ Document everything you learned
  • ☐ Send end-of-day summary to your manager
  • ☐ Plan your Day 2 priorities

Day 1 Success Criteria

  • ✓ You have working access to all systems
  • ✓ You know who to contact for questions
  • ✓ You understand current project status
  • ✓ You know your immediate priorities
  • ✓ You've documented the environment details
  • ✓ You've made a positive first impression

What NOT to Do on Day 1

Avoid These Mistakes
  • Don't make changes — Day 1 is for learning, not doing
  • Don't criticize existing work — You don't have context yet
  • Don't promise deliverables — You don't know what's feasible
  • Don't skip documentation — You will forget details
  • Don't be afraid to ask questions — It's expected on Day 1
  • Don't enable any searches in production — Wait until you understand the environment
