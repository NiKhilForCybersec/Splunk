Validation Commands to Run

Day 1 Validation SPL Commands
# 1. Check what indexes you can access
| eventcount summarize=false index=* | table index count

# 2. Check data recency for your index
index=YOUR_INDEX earliest=-24h | stats count by sourcetype | sort -count

# 3. Check field extraction
index=YOUR_INDEX | head 100 | fieldsummary | where count>0 | table field count

# 4. Check CIM compliance (e.g., for Web)
index=YOUR_INDEX | head 10 | table src, dest, action, http_method, url

# 5. Check data model acceleration
| rest /services/datamodel/acceleration | table title acceleration.enabled

# 6. Check installed apps
| rest /services/apps/local | search disabled=0 | table title version

# 7. Check your role/permissions
| rest /services/authentication/current-context | table username roles

Day 1 Printable Checklist

Morning Tasks

  • ☐ Arrive 15 minutes early
  • ☐ Set up laptop and verify network access
  • ☐ Get all required access (Splunk, email, Teams, documentation)
  • ☐ Meet team members and note names/roles
  • ☐ Get walkthrough of the environment
  • ☐ Identify your primary point of contact

Midday Tasks

  • ☐ Ask all environment questions (documented above)
  • ☐ Ask all project questions (documented above)
  • ☐ Review existing project documentation
  • ☐ Understand the current sprint/deliverables
  • ☐ Join all required meetings/channels

Afternoon Tasks

  • ☐ Log into Splunk and validate access
  • ☐ Run validation commands (above)
  • ☐ Review existing correlation searches
  • ☐ Document everything you learned
  • ☐ Send end-of-day summary to your manager
  • ☐ Plan your Day 2 priorities

Day 1 Success Criteria

  • ✓ You have working access to all systems
  • ✓ You know who to contact for questions
  • ✓ You understand current project status
  • ✓ You know your immediate priorities
  • ✓ You've documented the environment details
  • ✓ You've made a positive first impression

What NOT to Do on Day 1

Avoid These Mistakes
  • Don't make changes — Day 1 is for learning, not doing
  • Don't criticize existing work — You don't have context yet
  • Don't promise deliverables — You don't know what's feasible
  • Don't skip documentation — You will forget details
  • Don't be afraid to ask questions — It's expected on Day 1
  • Don't enable any searches in production — Wait until you understand the environment
