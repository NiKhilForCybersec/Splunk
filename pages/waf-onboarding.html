Prerequisites Checklist

Akamai SIEM Integration Steps

Step 1: Install Splunk Add-on for Akamai

In Splunk: Apps → Find More Apps → Search "Akamai" → Install "Splunk Add-on for Akamai SIEM" (TA-akamai)
Splunk Cloud: Submit support ticket with .tgz file if self-install disabled.

Step 2: Create Akamai API Credentials

In Akamai Control Center: Identity & Access → API Users → Create API Client
Required Scopes: SIEM, Events

Record these values:
Client ID: __________________________
Client Secret: ______________________
Access Token: _______________________
API Host: ___________________________ (e.g., akaa-xxxxx.luna.akamaiapis.net)

Step 3: Create Index in Splunk

Settings → Indexes → New Index
Name: akamai
Max Size: Based on retention requirements

Step 4: Configure Add-on Account

TA-akamai → Configuration → Add Account
Enter the API credentials from Step 2

Step 5: Create SIEM Input

TA-akamai → Inputs → Create New Input

SettingValue
Nameakamai_siem_events
Account(Select account from Step 4)
Config IDs(Security config IDs - ask Akamai admin)
Indexakamai
Interval60

Step 6: Validate Data Flow

Validation Query
index=akamai sourcetype=akamai:siem earliest=-1h
| stats count as events 
        dc(attackData.clientIP) as unique_ips
        latest(_time) as last_event
| eval last_event = strftime(last_event, "%Y-%m-%d %H:%M:%S")
| eval status = if(events > 0, "✅ Success", "❌ No data")

CyberArk Identity Onboarding

CyberArk Integration Method
CyberArk Identity typically uses SIEM connector or Syslog forwarding. Verify with CyberArk admin which method is configured.

Step 1: Install CyberArk Add-on

Splunkbase: "CyberArk Add-on for Splunk" or use generic syslog input

Step 2: Create Index

Settings → Indexes → New Index → Name: "cyberark"

Step 3: Configure SIEM Connector in CyberArk

CyberArk Admin Portal → Settings → SIEM Integration → Add Splunk endpoint

Step 4: Validate

index=cyberark sourcetype=cyberark:identity:audit earliest=-1h
| stats count dc(ActorId) as users dc(EventType) as events

Entra ID Onboarding

Step 1: Azure App Registration

Azure Portal → App registrations → New registration
Name: Splunk-EntraID-Integration
Record: Application (client) ID, Directory (tenant) ID

Step 2: Create Client Secret

Certificates & secrets → New client secret
Expiry: 24 months (max)
⚠️ Copy secret immediately - it won't be shown again

Step 3: Configure API Permissions

API permissions → Add a permission → Microsoft Graph → Application permissions
Add: AuditLog.Read.All, Directory.Read.All, Policy.Read.All
Click: Grant admin consent

Step 4: Install Microsoft Cloud Services Add-on

Splunk → Apps → Find More Apps → "Microsoft Cloud Services Add-on"

Step 5: Configure Add-on

Add-on → Configuration → Azure AD Account → Add
Enter: Client ID, Client Secret, Tenant ID

Step 6: Create Input

Inputs → Azure AD Sign-in Logs
Index: azure
Interval: 300 seconds

⚠️ Client Secret Expiry Alert
Set a calendar reminder 30 days before client secret expiry. Data ingestion will silently stop when it expires!

Questions to Ask Client
1. What retention do you need? (30d/90d/1yr)
2. Who needs access to search this data?
3. Is this PCI/HIPAA scope?
4. Current license utilization?
5. Index naming convention?

Data Validation Queries

Run These After Onboarding
``` Check data is flowing
index=akamai earliest=-1h | stats count by sourcetype

``` Check for timestamp issues
index=akamai earliest=+0d | stats count
``` If count > 0, you have future-dated events!

``` Check latency
index=akamai earliest=-1h
| eval latency = _indextime - _time
| stats avg(latency) as avg_sec max(latency) as max_sec

``` Find gaps in data
index=akamai earliest=-24h | bucket _time span=5m
| stats count by _time | where count=0
