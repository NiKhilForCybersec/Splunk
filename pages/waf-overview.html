Tier Definitions

Tier 1 — Foundation Detections

  • High confidence, low false positive rate
  • Detect obvious/volumetric attacks
  • Deploy first, establish baseline
  • Use cases: UC1-UC4

Tier 2 — Advanced Detections

  • Higher complexity, requires tuning
  • Detect evasive/sophisticated attacks
  • Deploy after Tier 1 is stable
  • Use cases: UC5-UC8

Detection Development Lifecycle

Phase 1: Data Validation (Day 1-2)

Confirm Akamai logs flowing, verify field extraction, check data freshness and latency.

Phase 2: Baseline Analysis (Day 3-5)

Calculate 14-day historical baseline for block volumes, identify normal patterns by hour/day.

Phase 3: Detection Development (Week 2-3)

Write SPL queries, test against historical data, tune thresholds using statistical methods.

Phase 4: FP Analysis (Week 3-4)

Run detections in silent mode, identify false positive sources, build exclusion lists.

Phase 5: Documentation (Week 4)

Create runbooks, dashboard panels, use case specifications for each detection.

Phase 6: Go-Live (Week 5+)

Enable alerts in production, monitor initial period, fine-tune as needed.
