WAF-UC3: Authentication Endpoint Attack

Attacks specifically targeting login, authentication, and password reset endpoints. Indicates brute force attempts or credential stuffing campaigns.

Severity HIGH
MITRE T1110
Schedule Every 5 min
Time Range Last 10 min

Step 1: Decision Tree (Detection Logic)

flowchart TD
    START[WAF Block Event Received] --> A{Does URI match auth endpoint pattern}
    A -->|No| OTHER[Route to other detection - not auth related]
    A -->|Yes| B[Event targets authentication endpoint]

    C --> D[Count total blocked requests]
    D --> E[Count unique attacker IPs]
    E --> F{Is blocked count GT 20}
    F -->|No| G{Is blocked GT 10 AND unique attackers GT 3}
    G -->|No| BELOW[Below threshold - continue monitoring]
    G -->|Yes| DISTRIBUTED[Distributed Attack Pattern - multiple sources]

    H -->|No| I{Is unique_attackers GT 3}
    I -->|Yes| COORDINATED[Coordinated Attack from multiple IPs]
    I -->|No| BRUTE[Single Source Brute Force Attack]
    STUFFING --> J{Is blocked count GT 100}
    J -->|Yes| CRIT[CRITICAL - Large scale credential stuffing]
    J -->|No| HIGH1[HIGH severity]

Prerequisite: Discover YOUR Auth Endpoints

Run This First to Find Authentication URIs in Your Data

Every environment has different auth endpoint paths. Run this discovery query to see what exists in YOUR data:

index=akamai sourcetype=akamai:siem earliest=-7d
| spath
| eval uri=coalesce(uri,'httpMessage.requestUri')
| stats count by uri
| search uri=*login* OR uri=*auth* OR uri=*sign* OR uri=*token* OR uri=*password* OR uri=*session* OR uri=*oauth*
| sort - count
| head 50

Output: List of URIs that look like auth endpoints with their request counts. Use this to customize the regex pattern in Step A.

Step A: Build SPL in Search First

Where: Splunk Search → Build and test SPL

Variant A: Standard Web Application Auth Endpoints

Use when: Standard web app login pages like /login, /signin, /auth. Customize the regex for YOUR endpoints. 

index=akamai sourcetype=akamai:siem earliest=-10m
| spath
| eval waf_action=coalesce(action, mvindex('attackData.rules{}.ruleAction',0))
| eval src_ip=coalesce(src_ip,'attackData.clientIP')
| eval uri=coalesce(uri,'httpMessage.requestUri')
| eval vhost=coalesce(vhost,'httpMessage.requestHeaders.Host')
| where waf_action="deny" OR waf_action="block"
```CUSTOMIZE THIS REGEX FOR YOUR AUTH ENDPOINTS:
| where match(uri, "(?i)(login|signin|sign-in|auth|oauth|token|session|password|forgot|reset|register|api/v[0-9]+/auth)")
| bin _time span=5m
| stats count as blocked 
        dc(src_ip) as unique_attackers 
        values(src_ip) as attacker_ips
        values(uri) as targeted_uris
        BY _time, uri, vhost
| where blocked > 20 OR (blocked > 10 AND unique_attackers > 3)
| eval attack_style = case(
    unique_attackers > 10, "Distributed Credential Stuffing",
    unique_attackers > 3, "Coordinated Attack",
    1=1, "Single Source Brute Force"
)
| eval severity = case(
    blocked > 100, "critical",
    blocked > 50, "high",
    1=1, "medium"
)
| table _time vhost uri blocked unique_attackers attacker_ips attack_style severity

Variant B: API-Based Authentication (OAuth, JWT, REST)

Use when: Modern API authentication endpoints like /api/v1/auth, /oauth/token, /jwt/refresh. 

index=akamai sourcetype=akamai:siem earliest=-10m
| spath
| eval waf_action=coalesce(action, mvindex('attackData.rules{}.ruleAction',0))
| eval src_ip=coalesce(src_ip,'attackData.clientIP')
| eval uri=coalesce(uri,'httpMessage.requestUri')
| eval vhost=coalesce(vhost,'httpMessage.requestHeaders.Host')
| eval method=coalesce(method,'httpMessage.requestMethod')
| where waf_action="deny" OR waf_action="block"
| where match(uri, "(?i)(/api/.*/auth|/oauth/|/token|/jwt/|/v[0-9]+/login|/v[0-9]+/authenticate|/api/login)")
| where method="POST"
| bin _time span=5m
| stats count as blocked 
        dc(src_ip) as unique_attackers 
        values(src_ip) as attacker_ips
        BY _time, uri, vhost
| where blocked > 20 OR (blocked > 10 AND unique_attackers > 3)
| eval attack_style = case(
    unique_attackers > 10, "Distributed API Auth Attack",
    unique_attackers > 3, "Coordinated API Attack",
    1=1, "Single Source API Brute Force"
)
| eval severity = case(blocked > 100, "critical", blocked > 50, "high", 1=1, "medium")
| table _time vhost uri blocked unique_attackers attacker_ips attack_style severity

Variant C: E-commerce / Shopping Cart Auth

Use when: Retail/e-commerce sites with checkout authentication, account creation, gift card validation. 

index=akamai sourcetype=akamai:siem earliest=-10m
| spath
| eval waf_action=coalesce(action, mvindex('attackData.rules{}.ruleAction',0))
| eval src_ip=coalesce(src_ip,'attackData.clientIP')
| eval uri=coalesce(uri,'httpMessage.requestUri')
| eval vhost=coalesce(vhost,'httpMessage.requestHeaders.Host')
| where waf_action="deny" OR waf_action="block"
| where match(uri, "(?i)(login|signin|checkout|account/create|register|giftcard|coupon|promo|cart/auth|my-account)")
| bin _time span=5m
| stats count as blocked 
        dc(src_ip) as unique_attackers 
        values(src_ip) as attacker_ips
        BY _time, uri, vhost
| where blocked > 20 OR (blocked > 10 AND unique_attackers > 3)
| eval attack_style = case(
    unique_attackers > 10, "Distributed Credential Stuffing",
    match(uri, "(?i)(giftcard|coupon|promo)"), "Gift Card / Promo Abuse",
    unique_attackers > 3, "Coordinated Attack",
    1=1, "Single Source Brute Force"
)
| eval severity = case(blocked > 100, "critical", blocked > 50, "high", 1=1, "medium")
| table _time vhost uri blocked unique_attackers attacker_ips attack_style severity

Step C: Drilldown Search

Show Attacker Details and Patterns

index=akamai sourcetype=akamai:siem earliest=-30m uri="$uri$" vhost="$vhost$"
| spath
| eval waf_action=coalesce(action, mvindex('attackData.rules{}.ruleAction',0))
| eval src_ip=coalesce(src_ip,'attackData.clientIP')
| eval status=coalesce(status,'httpMessage.status')
| eval user_agent=coalesce(http_user_agent,'httpMessage.requestHeaders.User-Agent')
| where waf_action="deny" OR waf_action="block"
| stats count as attempts 
        values(status) as http_codes
        values(user_agent) as user_agents
        latest(_time) as last_seen
        earliest(_time) as first_seen
        BY src_ip
| eval attack_duration_min = round((last_seen - first_seen) / 60, 1)
| eval attempts_per_min = round(attempts / if(attack_duration_min > 0, attack_duration_min, 1), 1)
| sort - attempts
| head 25

Step E: Handle False Positives

Common False Positives & Solutions

FP SourceHow to IdentifySolution
Legitimate password reset campaigns Spike correlates with IT communication about password expiry Coordinate with IT; suppress during known reset campaigns
SSO health checks SSO provider IPs (Okta, Azure AD, Ping) at regular intervals Allowlist SSO provider probe IP ranges
Mobile app background auth Mobile user agents, low but consistent volume Identify mobile app patterns; may need separate threshold
Service account automation Known service account IPs, programmatic patterns Allowlist known service account source IPs
Session timeout floods Spike in /session endpoints after maintenance window Correlate with maintenance calendar; temporary suppression
WAF rule too aggressive Many different users from diverse IPs getting blocked Review WAF rule tuning; may be blocking legitimate traffic
