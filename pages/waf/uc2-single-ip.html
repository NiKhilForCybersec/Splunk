WAF-UC2: Single Source Flood Attack

Detect when a single IP generates excessive requests - indicates brute force, credential stuffing, or denial-of-service attempt from a single source.

Default Severity HIGH
MITRE ATT&CK T1498.001
Schedule Every 5 min
Time Range Last 5 min

Step 1: Decision Tree (Detection Logic)

flowchart TD
    START[Aggregate all WAF events by src_ip over 5 min window] --> A[Calculate total_requests per source IP]
    A --> B[Calculate blocked_count per source IP]
    B --> C[Calculate unique_endpoints accessed per IP]
    C --> D{Is total_requests GT 500}
    D -->|No| E{Is blocked_count GT 100}
    E -->|No| BELOW[Below threshold - no alert needed]
    E -->|Yes| HIGH_BLOCK[High block volume from single IP]
    D -->|Yes| HIGH_VOL[High total volume from single IP]
    HIGH_BLOCK --> F[Calculate block_ratio percentage]
    HIGH_VOL --> F
    F --> F1[block_ratio = blocked_count divided by total_requests times 100]
    F1 --> G{Is block_ratio GT 90 percent}
    G -->|Yes| PURE[Pure Attack Pattern - almost all requests blocked]
    G -->|No| H{Is block_ratio GT 50 percent}
    H -->|Yes| MIXED[Mixed Traffic Pattern - suspicious activity]
    H -->|No| SUSPICIOUS[High volume but low block rate - investigate]
    PURE --> I{Is blocked_count GT 500}
    I -->|Yes| CRIT[CRITICAL severity - massive attack]
    I -->|No| J{Is blocked_count GT 200}

    HIGHSEV --> CREATE
    MEDSEV --> CREATE
    CREATE --> ENRICH[Enrich with target hosts and URIs]
    ENRICH --> THROTTLE[Throttle by src_ip for 30 minutes]
    THROTTLE --> ALERT[Send to Incident Review for analyst action]

Step A: Build SPL in Search First

Where: Splunk Search → Build and test SPL

A1 Basic Single IP Flood Detection

Use when: Simple single-IP flood detection. Good starting point for most environments. 

index=akamai sourcetype=akamai:siem earliest=-5m
| spath
| eval waf_action=coalesce(action, mvindex('attackData.rules{}.ruleAction',0))
| eval src_ip=coalesce(src_ip,'attackData.clientIP')
| eval vhost=coalesce(vhost,'httpMessage.requestHeaders.Host')
| eval uri=coalesce(uri,'httpMessage.requestUri')
| stats count as total_requests 
        count(eval(waf_action="deny" OR waf_action="block")) as blocked_count
        dc(uri) as unique_endpoints
        dc(vhost) as unique_hosts
        values(vhost) as target_hosts
        BY src_ip
| eval block_ratio = round(blocked_count / total_requests * 100, 1)
| where total_requests > 500 OR blocked_count > 100
| eval attack_type = case(
    block_ratio >= 90, "Pure Attack",
    block_ratio >= 50, "Mixed Traffic",
    1=1, "Suspicious Volume"
)
| eval severity = case(
    blocked_count > 500, "critical",
    blocked_count > 200, "high",
    1=1, "medium"
)
| table src_ip total_requests blocked_count block_ratio unique_endpoints unique_hosts target_hosts attack_type severity

A2 With Attack Signature Analysis

Use when: You want to know WHAT type of attacks the IP is attempting. 

index=akamai sourcetype=akamai:siem earliest=-5m
| spath
| eval waf_action=coalesce(action, mvindex('attackData.rules{}.ruleAction',0))
| eval src_ip=coalesce(src_ip,'attackData.clientIP')
| eval uri=coalesce(uri,'httpMessage.requestUri')
| eval signature=coalesce(signature,'attackData.rules{}.ruleTag')
| eval vhost=coalesce(vhost,'httpMessage.requestHeaders.Host')
| stats count as total_requests 
        count(eval(waf_action="deny" OR waf_action="block")) as blocked_count
        dc(uri) as unique_endpoints
        values(uri) as sample_uris
        values(signature) as attack_signatures
        values(vhost) as target_hosts
        BY src_ip
| eval block_ratio = round(blocked_count / total_requests * 100, 1)
| where total_requests > 500 OR blocked_count > 100
| eval is_auth_attack = if(match(mvjoin(sample_uris,","), "(?i)(login|auth|signin|password|oauth|token)"), "Yes", "No")
| eval is_injection_attack = if(match(mvjoin(attack_signatures,","), "(?i)(sql|xss|cmd|lfi|rfi)"), "Yes", "No")
| eval severity = case(
    blocked_count > 500, "critical",
    blocked_count > 200, "high",
    1=1, "medium"
)
| table src_ip total_requests blocked_count block_ratio unique_endpoints sample_uris attack_signatures is_auth_attack is_injection_attack target_hosts severity

A3 With Geographic Enrichment

Use when: You want geographic context about the attacker for threat intel and blocking decisions. 

index=akamai sourcetype=akamai:siem earliest=-5m
| spath
| eval waf_action=coalesce(action, mvindex('attackData.rules{}.ruleAction',0))
| eval src_ip=coalesce(src_ip,'attackData.clientIP')
| eval country=coalesce(src_country,'geo.country')
| eval city=coalesce(src_city,'geo.city')
| eval uri=coalesce(uri,'httpMessage.requestUri')
| stats count as total_requests 
        count(eval(waf_action="deny" OR waf_action="block")) as blocked_count
        dc(uri) as unique_endpoints
        values(country) as src_country
        values(city) as src_city
        BY src_ip
| eval block_ratio = round(blocked_count / total_requests * 100, 1)
| where total_requests > 500 OR blocked_count > 100
| iplocation src_ip
| eval severity = case(
    blocked_count > 500, "critical",
    blocked_count > 200, "high",
    1=1, "medium"
)
| table src_ip Country City total_requests blocked_count block_ratio unique_endpoints severity

Step C: Drilldown Search

C Show Detailed Attack Pattern for IP

index=akamai sourcetype=akamai:siem earliest=-15m src_ip="$src_ip$"
| spath
| eval waf_action=coalesce(action, mvindex('attackData.rules{}.ruleAction',0))
| eval uri=coalesce(uri,'httpMessage.requestUri')
| eval signature=coalesce(signature,'attackData.rules{}.ruleTag')
| eval status=coalesce(status,'httpMessage.status')
| eval user_agent=coalesce(http_user_agent,'httpMessage.requestHeaders.User-Agent')
| stats count as requests 
        values(waf_action) as actions
        values(status) as http_status
        values(user_agent) as user_agents
        by uri signature
| sort - requests
| head 30

Step E: Handle False Positives

E Common False Positives & Solutions

FP SourceHow to IdentifySolution
Corporate NAT/Proxy Known corporate egress IPs, many different user agents, diverse URI patterns Create corporate_egress.csv lookup with known NAT IPs, exclude or tag separately
Load testing tools Spikes during QA windows, JMeter/Gatling/Locust/k6 user agents Coordinate test schedules with QA, suppress during known windows
Monitoring services Pingdom/UptimeRobot/Datadog/New Relic IP ranges, regular intervals Allowlist monitoring service IP ranges in lookup
Search engine crawlers Googlebot/Bingbot user agents, verified crawler IPs Verify via reverse DNS, allowlist legitimate crawler IP ranges
CDN edge servers Requests from Akamai/Cloudflare/Fastly edge IPs forwarding traffic Use X-Forwarded-For header to get real client IP instead
Partner API integrations Known partner IPs, legitimate business traffic patterns Create partner_ips.csv lookup, exclude or create separate "partner anomaly" detection
