WAF-UC5: Distributed Low-and-Slow DDoS Detection

Detect when MANY different IPs each send FEW requests - classic botnet pattern that evades single-source detection. Each individual IP looks innocent, but together they form a coordinated attack.

Severity HIGH
MITRE T1498
Schedule Every 5 min
Time Range Last 15 min

How This Differs from UC2 (Single Source)

UC2: Single Source Flood

  • Pattern: 1 IP sends 1000+ requests
  • Detection: High volume per IP
  • Attacker: Single compromised machine or script
  • Evasion: Easy to detect and block

UC5: Distributed Low-and-Slow

  • Pattern: 500 IPs each send 5 requests
  • Detection: Many IPs, low per-IP volume
  • Attacker: Botnet, coordinated attack
  • Evasion: Evades per-IP rate limits

Key insight: If avg_requests_per_ip is LOW but total_blocks is HIGH, you have a distributed attack.

Step A: Build SPL in Search First

Where: Splunk Search → Build and test SPL

Variant A: Basic Distributed Detection

Use when: Starting out. Detects the distributed pattern based on IP count and per-IP average. 

index=akamai sourcetype=akamai:siem earliest=-15m
| spath
| eval waf_action=coalesce(action, mvindex('attackData.rules{}.ruleAction',0))
| eval src_ip=coalesce(src_ip,'attackData.clientIP')
| eval vhost=coalesce(vhost,'httpMessage.requestHeaders.Host')
| where waf_action="deny" OR waf_action="block"
| stats count as total_blocks 
        dc(src_ip) as unique_attackers
        values(src_ip) as sample_ips
        BY vhost
| where unique_attackers > 50 AND total_blocks > 200
| eval avg_per_ip = round(total_blocks / unique_attackers, 1)
| where avg_per_ip < 10
| eval botnet_size = case(
    unique_attackers > 500, "Massive Botnet (500+ IPs)",
    unique_attackers > 200, "Large Botnet (200-500 IPs)",
    unique_attackers > 100, "Medium Botnet (100-200 IPs)",
    1=1, "Small Distributed (50-100 IPs)"
)
| eval severity = case(
    unique_attackers > 500, "critical",
    unique_attackers > 100, "high",
    1=1, "medium"
)
| table vhost total_blocks unique_attackers avg_per_ip botnet_size sample_ips severity

Variant B: With Geographic Distribution Analysis

Use when: You want to understand WHERE the botnet is coming from geographically. 

index=akamai sourcetype=akamai:siem earliest=-15m
| spath
| eval waf_action=coalesce(action, mvindex('attackData.rules{}.ruleAction',0))
| eval src_ip=coalesce(src_ip,'attackData.clientIP')
| eval vhost=coalesce(vhost,'httpMessage.requestHeaders.Host')
| eval country=coalesce(src_country,'geo.country')
| where waf_action="deny" OR waf_action="block"
| iplocation src_ip
| stats count as total_blocks 
        dc(src_ip) as unique_attackers
        dc(Country) as unique_countries
        values(Country) as attack_countries
        BY vhost
| where unique_attackers > 50 AND total_blocks > 200
| eval avg_per_ip = round(total_blocks / unique_attackers, 1)
| where avg_per_ip < 10
| eval geo_spread = case(
    unique_countries > 20, "Global Botnet (20+ countries)",
    unique_countries > 10, "Multi-Region (10-20 countries)",
    unique_countries > 3, "Regional Cluster (3-10 countries)",
    1=1, "Single Region (1-3 countries)"
)
| eval severity = case(
    unique_attackers > 500, "critical",
    unique_attackers > 100 AND unique_countries > 10, "critical",
    unique_attackers > 100, "high",
    1=1, "medium"
)
| table vhost total_blocks unique_attackers avg_per_ip unique_countries geo_spread attack_countries severity

Variant C: With ASN/Network Analysis

Use when: You want to identify if attacks come from specific hosting providers, residential ISPs, or cloud providers. 

index=akamai sourcetype=akamai:siem earliest=-15m
| spath
| eval waf_action=coalesce(action, mvindex('attackData.rules{}.ruleAction',0))
| eval src_ip=coalesce(src_ip,'attackData.clientIP')
| eval vhost=coalesce(vhost,'httpMessage.requestHeaders.Host')
| eval asn=coalesce(src_asn,'attackData.asn','geo.asn')
| where waf_action="deny" OR waf_action="block"
| stats count as total_blocks 
        dc(src_ip) as unique_attackers
        dc(asn) as unique_asns
        values(asn) as top_asns
        BY vhost
| where unique_attackers > 50 AND total_blocks > 200
| eval avg_per_ip = round(total_blocks / unique_attackers, 1)
| where avg_per_ip < 10
| eval network_analysis = case(
    unique_asns < 5 AND unique_attackers > 100, "Concentrated in few networks - possible VPS/hosting abuse",
    unique_asns > 50, "Highly distributed across networks - likely residential botnet",
    1=1, "Mixed network distribution"
)
| eval severity = case(unique_attackers > 500, "critical", unique_attackers > 100, "high", 1=1, "medium")
| table vhost total_blocks unique_attackers avg_per_ip unique_asns network_analysis top_asns severity

Step C: Drilldown Search

Show Top Attacker IPs and Their Patterns

index=akamai sourcetype=akamai:siem earliest=-30m vhost="$vhost$"
| spath
| eval waf_action=coalesce(action, mvindex('attackData.rules{}.ruleAction',0))
| eval src_ip=coalesce(src_ip,'attackData.clientIP')
| eval uri=coalesce(uri,'httpMessage.requestUri')
| eval signature=coalesce(signature,'attackData.rules{}.ruleTag')
| where waf_action="deny" OR waf_action="block"
| iplocation src_ip
| stats count as requests
        dc(uri) as unique_uris
        values(uri) as sample_uris
        values(signature) as signatures
        values(Country) as country
        BY src_ip
| sort - requests
| head 50
| eval request_pattern = case(
    unique_uris > 20, "Scanner (hitting many endpoints)",
    unique_uris < 3, "Focused (same endpoints)",
    1=1, "Mixed"
)

Country Distribution Breakdown

index=akamai sourcetype=akamai:siem earliest=-30m vhost="$vhost$"
| spath
| eval waf_action=coalesce(action, mvindex('attackData.rules{}.ruleAction',0))
| eval src_ip=coalesce(src_ip,'attackData.clientIP')
| where waf_action="deny" OR waf_action="block"
| iplocation src_ip
| stats count as blocks dc(src_ip) as unique_ips BY Country
| sort - unique_ips
| head 20
| eval percentage = round(unique_ips / sum(unique_ips) * 100, 1)

Step E: Handle False Positives

Common False Positives & Solutions

FP SourceHow to IdentifySolution
Mobile carrier NAT Many IPs from same mobile carrier ASN, legitimate user agents Check ASN concentration; mobile carriers share IPs across many users
CDN/Proxy traffic IPs belong to CDN ranges (Cloudflare, Akamai edge) Exclude known CDN IP ranges; use X-Forwarded-For for real client
Corporate egress clusters Multiple IPs but same user agent pattern, same /24 subnet Check subnet clustering; corporate networks often have IP pools
Search engine crawlers Googlebot/Bingbot from many IPs, hitting many URIs Verify UA and reverse DNS; allowlist legitimate crawler ranges
Legitimate traffic spike Corresponds to marketing campaign, product launch Correlate with business events; may be WAF rules too aggressive
Security research/bug bounty Diverse IPs but from known security research ASNs Allowlist known bug bounty platform IPs if you have a program
