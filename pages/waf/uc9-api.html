WAF-UC9: API Endpoint Abuse

Excessive API requests indicating data scraping, enumeration, or abuse. API endpoints often expose sensitive data and are prime targets for automated extraction.

Severity CRITICAL
MITRE T1190
Schedule Every 5 min
Time Range Last 10 min

Why API Abuse is Critical

APIs expose structured data - Unlike web scraping HTML, API abuse extracts clean, structured data at scale.

Common targets: Customer data, pricing info, inventory levels, user enumeration, search functionality

Impact: Competitive intelligence theft, privacy violations, credential stuffing prep, pricing manipulation

Step A: Build SPL in Search First

Where: Splunk Search → Build and test SPL

API Abuse Detection

index=akamai sourcetype=akamai:siem earliest=-10m
| spath
| eval src_ip=coalesce(src_ip,'attackData.clientIP')
| eval uri=coalesce(uri,'httpMessage.requestUri')
| eval vhost=coalesce(vhost,'httpMessage.requestHeaders.Host')
| eval method=coalesce(method,'httpMessage.requestMethod')
| eval status=coalesce(status,'httpMessage.status')
```Filter to API endpoints - CUSTOMIZE FOR YOUR APIs:
| where match(uri, "(?i)(\/api\/|\/v[0-9]+\/|\/graphql|\/rest\/)")
| eval api_category = case(
    match(uri, "(?i)(user|profile|account|customer|member)"), "User/Account Data",
    match(uri, "(?i)(search|query|lookup|find)"), "Search Endpoint",
    match(uri, "(?i)(product|item|price|inventory|catalog)"), "Product/Pricing",
    match(uri, "(?i)(order|transaction|payment|cart)"), "Transaction Data",
    match(uri, "(?i)(list|index|all|export)"), "Listing/Export",
    1=1, "Other API"
)
| stats count as requests
        dc(uri) as unique_endpoints
        values(uri) as sample_uris
        values(api_category) as categories
        sum(eval(if(status>=200 AND status<300,1,0))) as successful
        BY src_ip, vhost
| where requests > 200 OR (requests > 50 AND match(mvjoin(categories,","), "(User|Customer|Transaction)"))
| eval success_rate = round(successful / requests * 100, 1)
| eval abuse_type = case(
    match(mvjoin(categories,","), "(User|Account|Customer)") AND requests > 100, "DATA SCRAPING - User/Customer records",
    match(mvjoin(categories,","), "Search") AND requests > 200, "SEARCH ABUSE - Exhaustive querying",
    match(mvjoin(categories,","), "Product") AND requests > 500, "PRICE SCRAPING - Product catalog",
    unique_endpoints < 3 AND requests > 500, "ENDPOINT HAMMERING - Single target abuse",
    1=1, "Generic API Abuse"
)
| eval severity = case(
    match(abuse_type, "(User|Customer|DATA)"), "critical",
    match(abuse_type, "(SEARCH|PRICE)"), "high",
    requests > 1000, "high",
    1=1, "medium"
)
| table src_ip vhost requests unique_endpoints success_rate categories abuse_type sample_uris severity

Response Actions

Immediate Actions for API Abuse

  1. Rate limit the source IP immediately at API gateway or WAF
  2. Check what data was accessed - review response sizes and content
  3. Identify if API key was used - if so, revoke it
  4. Check for data exfiltration - large response bodies = successful extraction
  5. Review API authentication - was this authenticated or anonymous access?
  6. Notify data/privacy team if customer data endpoints were scraped
