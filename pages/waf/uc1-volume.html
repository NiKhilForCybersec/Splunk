UC1: High Volume Blocked Requests from Single IP

Detection and investigation workflow for detecting potential DDoS or brute force attacks

Detection Summary

Threat:DDoS / Brute Force / Scanning
Data Source:Akamai SIEM API
MITRE ATT&CK:T1498 (Network DoS), T1110 (Brute Force)
Severity:Medium-High
Schedule:Every 5 minutes

Key Indicators

  • Single IP with >100 blocked requests in 5 min
  • Multiple WAF rules triggered
  • Consistent targeting of specific endpoints
  • Unusual user agent strings
  • High request rate from unexpected geos

Detection Logic

flowchart LR A[Akamai Events] --> B{action = blocked?} B -->|Yes| C[Group by src IP] C --> D{Count > threshold?} D -->|Yes| E[Generate Notable] D -->|No| F[No alert] B -->|No| F E --> G[Add risk score to IP] style A fill:#0096d6,color:#fff style E fill:#ef4444,color:#fff style F fill:#10b981,color:#fff

Correlation Search SPL

| tstats summariesonly=true count from datamodel=Web where Web.action="blocked" AND index=akamai by Web.src, Web.dest, Web.http_method span=5m | where count > 100 | rename Web.src as src, Web.dest as dest, Web.http_method as http_method | eval urgency=case( count > 1000, "critical", count > 500, "high", count > 100, "medium", 1=1, "low" ) | eval rule_name="WAF - High Volume Blocked Requests" | eval rule_description="Source IP ".src." had ".count." blocked requests in 5 minutes targeting ".dest

Threshold Guidelines

Blocked Count (5 min) Urgency Typical Cause
100-500 Medium Automated scanning, misconfigured app
500-1000 High Aggressive scan, targeted attack attempt
>1000 Critical Active DDoS, brute force attack

Investigation Queries

1. Deep Dive on Suspicious IP

index=akamai src="SUSPICIOUS_IP" earliest=-24h | stats count by action, http_method, http_uri, http_user_agent, rule_name | sort -count

2. Check for Successful Requests

index=akamai src="SUSPICIOUS_IP" action!="blocked" earliest=-24h | stats count by http_uri, status, action | sort -count

3. WAF Rules Triggered

index=akamai src="SUSPICIOUS_IP" action="blocked" earliest=-24h | stats count by rule_name, rule_id, rule_action | sort -count

4. Attack Pattern Analysis

index=akamai src="SUSPICIOUS_IP" earliest=-24h | timechart span=5m count by action | eval attack_intensity=blocked/allowed

5. Geographic Context

index=akamai src="SUSPICIOUS_IP" earliest=-24h | head 1 | iplocation src | table src, City, Country, lat, lon

Detection Tuning

Common False Positive Sources

FP Source How to Identify Tuning Action
Health check monitors Same user agent, regular intervals, same URI Add user agent or source to whitelist
Security scanners (authorized) Known scanner IPs, during testing windows Create time-based exception or IP whitelist
CDN/Proxy IPs Multiple users behind same IP Raise threshold for known proxy IPs
Misconfigured applications Internal IP, specific app user agent Fix app config; temporary whitelist

Threshold Adjustment

# Analyze normal baseline to set thresholds index=akamai action="blocked" earliest=-7d | timechart span=5m count by src | stats perc95(count) as p95, perc99(count) as p99, max(count) as max_count # Set threshold above p95 to reduce noise
