RB-MFA-01: MFA Fatigue Attack (Privileged)

Severity: Critical
MITRE: T1621
Source: CyberArk
SLA: 5 min triage
Confirm user is privileged (check role in alert)
Verify denial count ≥5 in 10-minute window
Check if success followed denials (CRITICAL indicator)
Note source IP(s) and location(s)
Pivot Query: User Activity
index=cyberark sourcetype=cyberark:identity:audit ActorId="[USER]" earliest=-1h
| stats count values(EventType) values(Result) values(ClientIP) BY _time
| sort _time
FindingClassificationAction
Denials only, no successAttack in ProgressContact user immediately
Denials followed by successLIKELY COMPROMISEDDisable account NOW
User confirms they were promptedConfirmed AttackReset credentials

If Attack in Progress (No Success)

  1. Call user on known phone number
  2. Warn: "Do NOT approve any MFA prompts"
  3. Monitor for success events
  4. Consider temporary account suspension

If Compromised (Success After Denials)

  1. IMMEDIATELY disable account
  2. Revoke all active sessions
  3. Force password reset
  4. Re-enroll MFA device
  5. Escalate to Incident Response
  6. Review privileged actions taken

RB-WAF-01: WAF Block Volume Anomaly

Severity: Medium
MITRE: T1190
Source: Akamai
SLA: 1 hr triage
Confirm block count exceeds threshold
Check if source is known scanner (Qualys, Tenable)
Verify not during planned security test
Identify top attacking IPs
Pivot Query: Top Sources
index=akamai sourcetype=akamai:siem earliest=-1h
| spath path=attackData.rules{}.ruleAction output=action
| where action="deny"
| spath path=attackData.clientIP output=src
| stats count values(httpMessage.path) as targets BY src
| sort -count | head 20
  • Known Scanner: Close as FP, add to allowlist if not present
  • Single Malicious IP: Verify blocked, document for threat intel
  • Coordinated Attack: Escalate to Tier 2, consider rate limiting
  • DDoS Indicators: Escalate immediately, engage CDN support
