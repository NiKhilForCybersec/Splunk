Splunk Cloud Limitations
Cannot directly edit props.conf/transforms.conf. All changes via: (1) Splunk Web UI, (2) Apps/Add-ons, or (3) Support tickets.

Detection Deployment Lifecycle

Phase 1: Development

Write SPL in Search, test against historical data, tune thresholds. Save as report (not alert yet).

Phase 2: Silent Mode

Schedule search to run every 5 min, output to lookup or summary index. NO alerts. Monitor for 1 week.

Phase 3: Limited Alerting

Enable alerts for CRITICAL severity only. Send to security team only. Monitor for FPs.

Phase 4: Full Production

Enable all severity levels. Add to Incident Review. Full SOC distribution.

Schedule*/5 * * * *Every 5 minutes for near real-time

Example savedsearches.conf
[WAF-UC1 Block Volume Anomaly]
search = index=akamai sourcetype=akamai:siem ... (SPL)
cron_schedule = */5 * * * *
dispatch.earliest_time = -10m
dispatch.latest_time = now
alert_type = number of events
alert_comparator = greater than
alert_threshold = 0
alert.suppress = 1
alert.suppress.period = 15m
alert.severity = 3
disabled = 0

Creating Lookup Files

Step 1: Prepare CSV

Create CSV with header row. Save as UTF-8 without BOM. Use lowercase field names.

Step 2: Upload to Splunk

Settings → Lookups → Lookup table files → Add new

Step 3: Create Lookup Definition

Settings → Lookups → Lookup definitions → Add new → Select the file

Step 4: Test

| inputlookup privileged_users.csv | head 5

Common Operations

Check Index Size

| rest /services/data/indexes 
| search title IN (akamai, cyberark, azure)
| table title currentDBSizeMB maxTotalDataSizeMB

List Scheduled Searches

| rest /services/saved/searches
| search is_scheduled=1 disabled=0
| table title cron_schedule next_scheduled_time
